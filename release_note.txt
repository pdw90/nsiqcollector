- 2.1.4 (10.03.31)
   * 강제로 bsaefilelist.txt 를 적용하지 않게 하는 --nodelta 옵션 추가
   
- 2.1.3 (09.11.13)
   * 일부 UTF-16으로 작성된 소스 코드의 경우 XML 결과 출력시 인식할 수 없는 Character로 인해 Invalid XML 이 출력되는 문제 해결
     
- 2.1 (09.09.06)
   * Delta 분석을 위해, LOC 분석시 filefilter.txt 만 적용한 항목은 Product 사이즈로, filefilter.txt와 
     basefilelist.txt 를 동시에 적용한 항목은 Delta 사이즈로 설정
   * 복잡도 분석 결과는 영향 없음. (기존대로 filefilter.txt와 basefilelist.txt 가 둘다 적용됨)
   * 리포팅 변경
     - CSV 출력시 마지막 컬럼에 Delta 여부 출력
     - HTML/Excel 출력시 Delta Summary 출력 및 각 파일행에 마지막에 Delta 여부 출력
     - XML 출력시 Delta 결과 추가
     - 화면 출력에 Product / Delta LOC 구분하여 출력

- 2.0.0 (09.06.12)

   * 주요 특징 
      - 자바 기반으로 재개발 (더이상 Python Interpreter 필요하지 않음)

      - 분석하기 전에 파일 필터링을 수행하여 분석 속도 개선 (약 3배 이상)
        * 기존은 일단 분석을 한후 리포트에서 제외하는 방식이나, 현재는 처음부터 분석을 실시하지 않음
        * Complexity와 Loc를 1PASS로 분석 수행

      - Legacy 코드 제외 기능 지원
        * -g 옵션을 사용하여 현재 디렉토리의 파일명 사이즈를 포함하는 basefilelist.txt 파일을 생성
        * 분석 대상 폴더에 basefilelist.txt 가 있을 경우, 이 파일을 읽어 현재 분석할 파일이 레가시 코드인지  판단함.
        * N'SIQ Collector는 파일명과 사이즈가 같을 경우 레가시 코드에서 변경이 없는 것으로 판단하고 분석 결과에서 제외함. 
        * N'SIQ Collector는 신규 / 변경된 파일에 대해서 추가로 파일필터 filefilter.txt를 사용하여 필터링 함.

      - 언어별 분포 출력
        * N'SIQ Collector 지원 언어별로 파일개수 / LOC / Code LOC를 화면에 출력함

      - Excel 리포트 출력 지원 
        * --excel 옵션 사용

      - 한글 폴더 명 지원 
      
      - 단일 바이너리로 설치 패키지 간략화
      
      - CodeAnalyzerPro 실행을 위해 더이상 Temp 폴더 생성하지 않음
      
      - Verbose 기능 강화
        * -v 옵션을 사용하면 현재 분석중인 파일명이 출력됨. 따라서 분석 Hang시 문제가 생긴 파일을 쉽게 파악 가능
